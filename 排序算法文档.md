|------|----|

**已实现的排序算法：**

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 | 特殊渲染 |
|------|-----------|-----------|--------|---------|
| 冒泡排序 | O(n²) | O(1) | 稳定 | 单排柱状图 |
| 插入排序 | O(n²) | O(1) | 稳定 | 单排柱状图 |

| 桶排序 | O(n + k) | O(n + k) | 稳定 | 分桶区 + 飞行动画 |
| 堆排序 | O(n log n) | O(1) | 不稳定 | 二叉树小球 + 弧线动画 |

## 3. 核心架构

### 3.1 四阶段管线设计

整个系统围绕 **"语义步骤 → 时间轴步骤 → 插值帧 → Canvas 绘制"** 四阶段管线：

```
┌──────────────────────────────────────────────────────────────────┐
│  Stage 1: 排序算法（纯函数）                                      │
│  sortingAlgorithms.ts                                            │
│  输入: number[]                                                  │
│  输出: SemanticStep[]                                            │
│  职责: 描述排序逻辑（比较、交换、标记已排序等）                       │
└──────────────────────────┬───────────────────────────────────────┘
                           │ SemanticStep[]
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│  Stage 2: 时间轴构建器（layout + 状态追踪）                        │
│  timeline-builders/                                              │
│  输入: SemanticStep[] + 布局参数                                  │
│  输出: TimelineStep[]                                            │
│  职责: 为每一步生成 from/to FrameState + Transition 动画元信息     │
└──────────────────────────┬───────────────────────────────────────┘
                           │ TimelineStep[]
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│  Stage 3: 时间轴播放器 + 帧插值                                   │
│  useTimelinePlayer.ts + frame/                                   │
│  输入: TimelineStep[] + progress(0~1)                            │
│  输出: FrameState（当前帧的完整画面描述）                           │
│  职责: RAF 驱动时间推进，对 from/to 做实体级插值                    │
└──────────────────────────┬───────────────────────────────────────┘
                           │ FrameState
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│  Stage 4: Canvas 渲染器                                          │
│  useCanvasRenderer.ts                                            │
│  输入: FrameState                                                │
│  输出: 像素（Canvas 2D 绘制）                                     │
│  职责: 按 zIndex 绘制实体、叠加层、背景网格                        │
└──────────────────────────────────────────────────────────────────┘
```

## 4. 类型系统

### 4.1 SemanticStep — 语义步骤

排序算法生成的核心数据结构，描述"发生了什么"：

```typescript
// src/types/timeline.ts
interface SemanticStep {
  type: SemanticStepType;           // 步骤类型
  indices: number[];                // 涉及的数组下标
  description: string;              // 步骤描述（UI 显示）
  arraySnapshot?: number[];         // 操作后的数组快照
  gap?: number;                     // 步长（希尔排序）
  groupIndices?: number[];          // 当前排序组的全部下标
  tempSnapshot?: (number | null)[]; // 临时快照（归并排序）
  bucketIndex?: number;             // 目标桶编号（桶排序）
  bucketPos?: number;               // 桶内目标位置（桶排序）
}
```

### 4.2 SemanticStepType 完整列表

| StepType | 触发算法 | 含义 |
|----------|---------|------|
| `compare` | 所有 | 比较两个元素 |
| `swap` | 冒泡/快速/希尔/插入 | 交换两个元素 |
| `sorted` | 所有 | 标记元素为已排序 |
| `pivot` | 快速排序/堆排序 | 标记基准/当前 sift-down 根 |
| `set` | 插入/希尔 | 写入一个元素 |
| `merge` | — | 预留 |
| `merge-set` | 归并排序 | 胜出元素飞入下排辅助数组 |
| `merge-back` | 归并排序 | 辅助数组整段飞回上排 |
| `bucket-scatter` | 桶排序 | 元素从主数组飞入桶 |
| `bucket-compare` | 桶排序 | 桶内两元素比较 |
| `bucket-swap` | 桶排序 | 桶内两元素交换 |
| `bucket-gather` | 桶排序 | 元素从桶飞回主数组归位 |

### 4.3 RenderableEntity — 可渲染实体

屏幕上每个可视元素都是一个 `RenderableEntity`，通过稳定 `id` 在帧间匹配：

```typescript
interface RenderableEntity {
  id: string;            // 稳定标识符（帧间匹配用）
  sourceId: string;      // 来源标识（对应原始数组下标）

  value: number;         // 数值

```

### 4.5 StateTag — 状态标签

|-----|------|---------|

| `swapping` | 红色 `#ff5c5c` | 交换中 |

## 5. 排序算法详解

所有算法位于 `src/utils/sortingAlgorithms.ts`，均为纯函数，接收 `number[]` 返回 `SemanticStep[]`。

### 5.1 冒泡排序

**思想**：相邻元素两两比较，大值不断"冒泡"到数组末端。

```typescript
for (let i = 0; i < n - 1; i++) {
  for (let j = 0; j < n - i - 1; j++) {
    // compare [j, j+1]
    if (array[j] > array[j + 1]) // swap [j, j+1]
  }
  // sorted [n-i-1]
}
```

**步骤序列**：`compare → (swap?) → compare → ... → sorted`（每轮末尾一个 sorted）

---

### 5.2 插入排序

**思想**：将数组分为有序区和无序区，逐个将无序区元素插入有序区。

实现上使用真正的 `swap` 步骤来表达"key 向左移动"，使动画视觉更直观。

```typescript
for (let i = 1; i < n; i++) {
  let j = i;
  while (j >= 1 && array[j-1] > array[j]) {
    // swap [j-1, j]
    j--;
  }
}
```

---

### 5.3 快速排序

**思想**：选择基准（末位元素），Lomuto 分区，递归排序左右两段。

```typescript
function partition(low, high) {
  // pivot [high]
  for j = low to high-1:
    // compare [j, high]
    if array[j] < pivot: // swap [i++, j]
  // swap [i+1, high]（基准归位）
  // sorted [i+1]
}
```

---

### 5.4 希尔排序

**思想**：插入排序的改进，初始 `gap = n/2`，每轮减半，对每组间隔为 `gap` 的子序列做插入排序。

步骤携带 `gap` 和 `groupIndices`（当前组的所有下标），时间轴构建器用 `groupIndices` 设置 `pending` 状态标签。

```typescript
for (let gap = Math.floor(n / 2); gap > 0; gap = Math.floor(gap / 2)) {
  for (let i = gap; i < n; i++) {
    // 对 array[i] 所在子组做插入排序（compare + swap，携带 gap/groupIndices）
  }
}
```

---

### 5.5 归并排序

**思想**：分治，递归分割为单元素，自底向上两路归并。

**双排动画步骤序列**：

```
sort(left, mid)                         ← 递归左半
sort(mid+1, right)                      ← 递归右半
compare[left, right] + groupIndices     ← 开始合并，高亮整个区间
loop:
  compare[i, j] + groupIndices          ← 比较左右指针
  merge-set[source, dest]               ← 胜出元素飞入下排
merge-back[range] + arraySnapshot       ← 下排整段飞回上排
sorted[all]                             ← 全部完成
```

**关键设计**：
- `merge-set` **不携带** `arraySnapshot`，避免触发主数组重建
- `merge-back` 携带完整快照，触发主数组重建
- 上排飞走的位置渲染为幽灵实体（`ghost-bar`，opacity 0.4）

---

### 5.6 桶排序

**思想**：将元素按值域均匀分配到 K 个桶中，对每桶内部做插入排序，再依次收集归位。

**动态桶数公式**（`src/types/sorting.ts`）：

```typescript
export function calcBucketCount(n: number): number {
  return Math.min(9, Math.max(3, Math.round(n / 10)));
}
```

| 元素数 n | 桶数 K |
|---------|-------|
| 10 ~ 34 | 3 |
| 35 ~ 44 | 4 |
| 45 ~ 54 | 5 |
| 55 ~ 64 | 6 |
| 65 ~ 74 | 7 |
| 75 ~ 84 | 8 |
| 85 ~ 100 | 9 |

**分桶公式**（值域均匀分配）：
```
bi = min(K-1, floor(((value - minV) / range) * K))
```

**步骤序列**：
```
bucket-scatter × n                      ← 所有元素依次飞入对应桶
（对每个非空桶做插入排序）
  bucket-compare × m                    ← 桶内比较
  bucket-swap × m                       ← 桶内交换
bucket-gather × n                       ← 所有元素依次飞回归位
sorted[all]                             ← 完成
```

---

### 5.7 堆排序

**思想**：利用完全二叉树（堆）的性质，分两个阶段完成排序：

1. **建堆阶段**（Floyd 算法，O(n)）：从最后一个非叶子节点向根逐个执行 `sift-down`
2. **提取阶段**（O(n log n)）：反复将堆顶与末尾元素互换并就位，再对新根执行 `sift-down`

支持**最大堆**（升序）和**最小堆**（降序）两种模式：

```typescript
export function heapSort(arr: number[], mode: "max" | "min" = "max"): SemanticStep[] {
  const shouldSwap = (parent, child) =>
    mode === "max" ? a[parent] < a[child] : a[parent] > a[child];
}
```

**步骤序列**：

```
建堆阶段（从 i = floor(n/2)-1 到 0）：
  pivot[i]                        ← 标记当前 sift-down 根节点
  loop:
    compare[target, child]        ← 与左/右子比较
    swap[root, target]            ← 若需要，交换并继续向下

提取阶段（end = n-1 到 1）：
  swap[0, end]                    ← 堆顶飞到末尾
  sorted[end]                     ← 末尾元素就位
  （重复 sift-down 修复根）
sorted[0]                         ← 最后一个元素就位
```

**`groupIndices`**：每次 sift-down 期间，将当前有效堆范围 `[0, end]` 全部加入 `groupIndices`，渲染为 `pending` 状态，直观显示当前堆区域。

## 8. 帧插值系统

帧插值系统位于 `src/utils/frame/`，负责在 `from` 和 `to` 两个 `FrameState` 之间平滑过渡。

### 8.1 插值流程

```
TimelineStep + progress(0~1)
    │
    ▼
interpolateFrame()
    ├── applyEasing(progress)              ← 应用缓动函数
    └── interpolateEntities(from, to, transition, easedProgress)
        └── 对每个 to.entity:
            ├── 找到 from 中的匹配 entity（by id）
            ├── 如果 transition.type === "instant" → 直接返回 to
            └── 否则 interpolateEntity():
                ├── 位置：lerp(x, y)
                ├── arc 类型 → getArcPoint()（正弦弧线）
                ├── path 类型 → getPathPoint()（L 形折线）
                ├── 尺寸：lerp(width, height)
                ├── 透明度：lerp 或 getFadeOpacity()
                └── 样式：interpolateStyle()
```

### 8.2 缓动函数

```typescript
// 线性
lerp(start, end, progress) = start + (end - start) * progress

// 缓出（先快后慢）
easeOutCubic(progress) = 1 - (1 - progress)³

// 缓入缓出
easeInOutCubic(progress) =
  progress < 0.5 ? 4p³ : 1 - (-2p + 2)³ / 2
```

### 8.3 弧线路径（arc）

正弦弧线，用于 swap 和桶排序飞入飞出：

```typescript
getArcPoint(start, end, progress, arcHeight) = {
  x: lerp(start.x, end.x, progress),
  y: lerp(start.y, end.y, progress) - sin(progress * π) * arcHeight
}
```

`arcHeight` 取 `max(60, 平均高度 * 1.2)`。

### 8.4 L 形路径（path）

两段折线 + 弧度，用于归并排序上下排飞行：

```
horizontal-first 模式:          vertical-first 模式:
  起点                            起点
    ┌──────┐                        │
    │      │ (弧度)                  │ (弧度)
    │      └──→ 终点                 └──────→ 终点
```

- 前半段：`lerp → mid` + `sin(localProgress * π) * curveHeight`
- 后半段：`mid → end` + `sin((1-localProgress) * π) * curveHeight * 0.35`
- `curveHeight` 默认 48px

### 8.5 样式插值

```typescript
interpolateStyle(from, to, progress) = {
  fill:   progress < 0.5 ? from.fill   : to.fill,    // 50% 处切换
  stroke: progress < 0.5 ? from.stroke : to.stroke,
  text:   progress < 0.5 ? from.text   : to.text,
  glow:   lerp(from.glow, to.glow, progress),         // 连续插值
  dashed: progress < 0.5 ? from.dashed : to.dashed,
  alpha:  lerp(from.alpha, to.alpha, progress),        // 连续插值
}
```

## 10. 动画速度控制

### 速度参数传递链

```
ControlPanel 滑块（min: 20ms，max: 500ms）
    → sortStore.animationSpeed
    → 算法页面 props.speed
    → useSortAnimation.speed
    → rebuild() 重新生成 TimelineStep[]（stepDuration = speed.value）
    → useTimelinePlayer 使用 step.duration 驱动 RAF 循环
```

**速度变化时**：`useSortAnimation` 监听 `speed` 变化，触发 `rebuild()` 重新生成整条时间轴，`player.reset()` 重置到起始位置。

### useTimelinePlayer 播放引擎

```typescript
// src/composables/useTimelinePlayer.ts
export function useTimelinePlayer(steps: () => TimelineStep[]) {
  // 状态
  currentStepIndex: Ref<number>   // 当前步骤索引
  progress: Ref<number>           // 步内进度 0~1
  isPlaying: Ref<boolean>         // 是否正在播放
  currentFrame: ComputedRef       // 当前帧（通过 interpolateFrame 计算）

  // 方法
  play()              // 连续播放（RAF 循环）
  pause()             // 暂停
  stepForward()       // 单步前进（带动画）
  renderCurrentStep() // 跳到当前步末尾（无动画）
  reset()             // 重置到起始
}
```

**RAF 播放循环**：
- 计算 `elapsed = now - stepStartedAt`
- `progress = min(1, elapsed / step.duration)`
- `progress >= 1` 时自动推进到下一步
- 到达末尾时自动 `pause()`

### isAnimating 保护

`useTimelinePlayer` 通过 RAF 循环天然避免了并发问题——同一时间只有一个 `tick` 在运行。`play()` 入口检查 `isPlaying` 和 `rafId` 防止重复启动。

## 附录：设计模式总结

### Entity-Based 渲染

所有屏幕元素都是 `RenderableEntity`，通过稳定 `id` 在帧间匹配。插值系统根据 `id` 找到对应的 `from` 和 `to` 实体，实现平滑过渡。

### StateTag 驱动样式

实体携带 `stateTags[]`，渲染器通过 `getStyleFromStateTags()` 映射为视觉样式。算法语义与渲染细节完全解耦。

### Ghost 实体

归并排序和桶排序在元素飞行时，在源位置创建 `ghost-bar` 实体（opacity 0.4），形成视觉残留效果。幽灵实体在飞行结束后消失。

### displayIndex 追踪

每个元素有固定的 `displayIndex`（1-based），交换时随元素一起移动。动画中显示 `displayIndex` 可以直观看到元素的来源位置。

### 渲染器统一化

三个专用渲染器（`useBucketSortRenderer`、`useHeapSortRenderer`、`useMergeSortRenderer`）都是 `useCanvasRenderer` 的薄封装。视觉差异完全由时间轴构建器决定——它们生成不同的 `EntityKind`、`RenderableRegion` 和 `RenderableOverlay`。
