## 3. 核心架构

### 3.1 三层分离设计

整个系统围绕 **"步骤生成 → 步骤消费 → Canvas 绘制"** 三层分离：

```
┌──────────────────────────────────────────────────────────────┐
│                      用户操作层                               │
│  ControlPanel（速度/大小/算法）→ sortStore（Pinia）           │
└──────────────────────┬───────────────────────────────────────┘
                       │ algorithm, animationSpeed, originalArray
                       ▼
┌──────────────────────────────────────────────────────────────┐
│                   算法页面层（algorithms/）                    │
│  BubbleSort.vue / MergeSort.vue / BucketSort.vue ...         │
│  调用 useSortAnimation，持有 ISortCanvas 引用                 │
└──────────┬────────────────────────────────────────┬──────────┘
           │ sortFn(arr) → SortStep[]               │ canvasRef.applyStep(step)
           ▼                                        ▼
┌──────────────────────┐             ┌──────────────────────────┐
│  sortingAlgorithms   │             │    Canvas 组件            │
│  （步骤生成，纯函数） │             │  SortBarCanvas            │
│  bubbleSort()        │             │  SortBarCanvasMerge       │
│  mergeSort()         │             │  SortBarCanvasBucket      │
│  bucketSort() ...    │             └──────────┬───────────────┘
└──────────────────────┘                        │
                                                ▼
                                  ┌─────────────────────────────┐
                                  │    渲染器（composables）     │
                                  │  useCanvasRenderer           │
                                  │  useMergeSortRenderer        │
                                  │  useBucketSortRenderer       │
                                  │  requestAnimationFrame 循环  │
                                  └─────────────────────────────┘
```

## 4. 数据流与类型定义

### 4.1 SortStep — 步骤描述对象

所有算法通过生成 `SortStep[]` 描述排序过程，渲染器消费步骤驱动动画：

```typescript
interface SortStep {
  type: StepType;            // 步骤类型（见下表）
  indices: number[];         // 涉及的数组下标
  description: string;       // 步骤描述（UI 显示）
  arraySnapshot?: number[];  // 操作后的数组快照（用于重建 array.value）
  gap?: number;              // 步长（希尔排序专用）
  groupIndices?: number[];   // 当前排序组的全部下标（归并/快速用）
  bucketIndex?: number;      // 目标桶编号（桶排序专用）
  bucketPos?: number;        // 桶内目标位置（桶排序专用）
}
```

### 4.2 StepType 完整列表

| StepType | 触发算法 | 含义 |
|----------|---------|------|
| `compare` | 所有 | 比较两个元素 |
| `swap` | 冒泡/快速/希尔/插入 | 交换两个元素（带弧线动画） |
| `sorted` | 所有 | 标记元素为已排序（绿色） |
| `pivot` | 快速排序 | 标记基准元素（紫色） |
| `set` | 插入/希尔 | 写入一个元素（不对称交换） |
| `merge` | — | 预留（归并用 merge-set/back） |
| `merge-set` | 归并排序 | 胜出元素飞入下排辅助数组 |
| `merge-back` | 归并排序 | 辅助数组整段飞回上排 |
| `bucket-scatter` | 桶排序 | 元素从主数组飞入桶 |
| `bucket-compare` | 桶排序 | 桶内两元素比较（黄色高亮） |
| `bucket-swap` | 桶排序 | 桶内两元素交换（红色高亮） |
| `bucket-gather` | 桶排序 | 元素从桶飞回主数组归位 |

### 4.3 ArrayElement — 数组元素

```typescript
interface ArrayElement {
  value: number;        // 数值（1 ~ n，无重复）
  displayIndex: number; // 固定序号（1-based），随元素一起移动，不随位置变化
}
```

`displayIndex` 的作用：在动画中标识"这是初始第几个位置的元素"，可视化元素的来源位置。

### 4.4 ARRAY_MUTATING_TYPES — 只有这些步骤才重建数组

```typescript
const ARRAY_MUTATING_TYPES = new Set<SortStep['type']>(['swap', 'merge', 'set', 'merge-back']);
```

`compare`、`pivot`、`sorted`、`merge-set`、所有 `bucket-*` 步骤均**不触发** `array.value` 重建，避免误清空动画状态。

## 5. 排序算法详解

所有算法位于 `src/utils/sortingAlgorithms.ts`，均为纯函数，接收 `number[]` 返回 `SortStep[]`。

### 5.1 冒泡排序

**思想**：相邻元素两两比较，大值不断"冒泡"到数组末端。

```typescript
for (let i = 0; i < n - 1; i++) {
  for (let j = 0; j < n - i - 1; j++) {
    // compare [j, j+1]
    if (array[j] > array[j + 1]) // swap [j, j+1]
  }
  // sorted [n-i-1]
}
```

**步骤序列**：`compare → (swap?) → compare → ... → sorted`（每轮末尾一个 sorted）

---

### 5.2 插入排序

**思想**：将数组分为有序区和无序区，逐个将无序区元素插入有序区。

实现上使用真正的 `swap` 步骤（而非 `set`）来表达"key 向左移动"，使动画视觉更直观。

```typescript
for (let i = 1; i < n; i++) {
  let j = i;
  while (j >= 1 && array[j-1] > array[j]) {
    // swap [j-1, j]
    j--;
  }
}
```

---

### 5.3 快速排序

**思想**：选择基准（末位元素），Lomuto 分区，递归排序左右两段。

```typescript
function partition(low, high) {
  // pivot [high]
  for j = low to high-1:
    // compare [j, high]
    if array[j] < pivot: // swap [i++, j]
  // swap [i+1, high]（基准归位）
  // sorted [i+1]
}
```

---

### 5.4 希尔排序

**思想**：插入排序的改进，初始 `gap = n/2`，每轮减半，对每组间隔为 `gap` 的子序列做插入排序。

步骤携带 `gap` 和 `groupIndices`（当前组的所有下标），渲染器用 `groupIndices` 高亮 pending 状态。

```typescript
for (let gap = Math.floor(n / 2); gap > 0; gap = Math.floor(gap / 2)) {
  for (let i = gap; i < n; i++) {
    // 对 array[i] 所在子组做插入排序（compare + swap，携带 gap/groupIndices）
  }
}
```

---

### 5.5 归并排序

**思想**：分治，递归分割为单元素，自底向上两路归并。

**双排动画步骤序列**：

```
sort(left, mid)                         ← 递归左半
sort(mid+1, right)                      ← 递归右半
compare[left, right] + groupIndices     ← 开始合并，上排高亮整个区间
loop:
  compare[i, j] + groupIndices          ← 比较左右指针（黄色）
  merge-set[source, dest]               ← 胜出元素飞入下排（source=上排索引，dest=下排列号）
merge-back[range] + arraySnapshot       ← 下排整段飞回上排（携带合并后快照）
sorted[all]                             ← 全部完成
```

**关键设计**：
- `merge-set` **不携带** `arraySnapshot`（主数组未变），避免触发 `updateBars` 清空飞行中的下排柱子
- `merge-back` 携带完整快照，触发主数组重建
- 上排飞走的位置渲染为幽灵（半透明虚线占位）

---

### 5.6 桶排序


**思想**：将元素按值域均匀分配到 K 个桶中，对每桶内部做插入排序，再依次收集归位。

**动态桶数公式**（`src/types/sorting.ts` 中定义，算法层和渲染层共同 import）：

```typescript
export function calcBucketCount(n: number): number {
  return Math.min(9, Math.max(3, Math.round(n / 10)));
}
```

| 元素数 n | 桶数 K |
|---------|-------|
| 10 ~ 34 | 3 |
| 35 ~ 44 | 4 |
| 45 ~ 54 | 5 |
| 55 ~ 64 | 6 |
| 65 ~ 74 | 7 |
| 75 ~ 84 | 8 |
| 85 ~ 100 | 9 |

**分桶公式**（值域均匀分配）：
```
bi = min(K-1, floor(((value - minV) / range) * K))
```

**步骤序列**：
```
bucket-scatter × n                      ← 所有元素依次飞入对应桶
（对每个非空桶做插入排序）
  bucket-compare × m                    ← 桶内比较
  bucket-swap × m                       ← 桶内交换
bucket-gather × n                       ← 所有元素依次飞回归位
sorted[all]                             ← 完成
```

### 5.7 堆排序

**思想**：利用完全二叉树（堆）的性质，分两个阶段完成排序：

1. **建堆阶段**（Floyd 算法，O(n)）：从最后一个非叶子节点向根逐个执行 `sift-down`，将无序数组原地建成堆
2. **提取阶段**（O(n log n)）：反复将堆顶（最大/最小值）与末尾元素互换并就位，再对新根执行 `sift-down` 修复堆性质

支持**最大堆**（升序排列）和**最小堆**（降序排列）两种模式，通过比较器方向切换：

```typescript
export function heapSort(arr: number[], mode: "max" | "min" = "max"): SortStep[] {
  // 最大堆：父 >= 子；最小堆：父 <= 子
  const shouldSwap = (parent, child) =>
    mode === "max" ? a[parent] < a[child] : a[parent] > a[child];
}
```

**两个阶段的 sift-down 起点区别**：

| 阶段 | sift-down 起点 | 说明 |
|------|--------------|------|
| 建堆（Floyd） | 最后非叶子节点 `floor(n/2)-1` 向根逐个 | 每个节点起点不同，O(n) 建堆 |
| 提取 | 每次都是根节点 `0` | 堆顶换至末尾后，新根需要向下修复 |

**步骤序列**：

```
建堆阶段（从 i = floor(n/2)-1 到 0）：
  pivot[i]                        ← 标记当前 sift-down 根节点（紫色）
  loop:
    compare[target, child]        ← 与左/右子比较（黄色，heapRange=pending 高亮当前堆范围）
    swap[root, target]            ← 若需要，交换并继续向下

提取阶段（end = n-1 到 1）：
  swap[0, end]                    ← 堆顶飞到末尾
  sorted[end]                     ← 末尾元素就位（绿/青）
  （重复 sift-down 修复根）
sorted[0]                         ← 最后一个元素就位，排序完成
```

**`groupIndices`（`heapRange`）**：每次 sift-down 期间，将当前有效堆范围 `[0, end]` 全部加入 `groupIndices`，渲染为 `pending`（淡紫色小球），直观显示当前还在参与排序的堆区域。

### 6.2 归并排序渲染器（useMergeSortRenderer）

**布局**：上排主数组 + 下排辅助数组（merge buffer）：

```
┌──────────────────────────────────┐  ← 上排（主数组）
│  ██  ██  ██  ██  ██  ██  ██     │    幽灵占位（已飞走的位置）用虚线显示
└──────────────────────────────────┘
    ↓ merge-set（飞入）    ↑ merge-back（飞回）
┌──────────────────────────────────┐  ← 下排（辅助数组 / 合并缓冲区）
│        ██  ██  ██                │    橙色=最新，青色=已在下排
└──────────────────────────────────┘
```

**动画时长系数**：

| 动画阶段 | 系数 | 说明 |
|---------|------|------|
| 飞入下排（merge-set） | × 0.8 | 稍快，强调胜者快速产生 |
| 飞回上排（merge-back） | × 1.2 | 稍慢，强调合并完成感 |
| 交换（swap，兼容预留） | × 1.0 | 标准 |

**下排柱子颜色**：

| 颜色 | 含义 |
|------|------|
| 橙色 `rgb(255,160,50)` | 最新飞入的元素（`isLatest=true`） |
| 青色 `rgb(78,205,196)` | 已落入的旧元素（`isLatest=false`） |
| 绿色 `rgb(103,194,58)` | 即将飞回上排（merge-back 时） |

### 6.3 桶排序渲染器（useBucketSortRenderer）

**布局**：三区垂直分割：

```
│  ██ ██ [虚线] ██ ██ ██ [虚线] ██  ← 已飞走的显示虚线  │

├──────┬──────┬──────┬──────┬──────────────────────────┤

│ ██   │ ██   │      │ ██   │                           │
└──────┴──────┴──────┴──────┴──────────────────────────┘
```

**桶标题栏内容**：
- 左上角：`B0`、`B1`... 桶编号（小字）
- 居中：`桶 N`（大字）+ `[ rS – rE ]`（值域范围）
- 右上角：当前桶内元素数徽章

**桶内柱子宽度防溢出（两步法）**：

```
maxPerBucket = ceil(n / K)
available    = BKT_W - BB_PAD * 2

# 先尝试正常间距 BB_G = 6
BB_W = floor((available + BB_G) / maxPerBucket - BB_G)

# 若宽度不足 4px，收窄间距至 2
if BB_W < 4:
    BB_G = 2
    BB_W = floor((available + BB_G) / maxPerBucket - BB_G)

BB_W = clamp(BB_W, 2, 48)
```

保证 `maxPerBucket × (BB_W + BB_G) - BB_G ≤ available`，桶内柱子始终不溢出桶框。

**背景网格离屏缓存**：网格是静态内容，使用离屏 `HTMLCanvasElement` 缓存，每帧 `drawImage` 贴图，避免每帧重绘 `O(W/40 + H/40)` 条线段。尺寸变化时自动重建缓存。

### 6.4 堆排序渲染器（useHeapSortRenderer）

**布局**：完全二叉树节点布局 + 底部数组行：

```
┌──────────────────────────────────────────────────────────────┐
│                       TREE 区                                 │
│               ●                ← 根节点（深度 0）              │
│          ●         ●           ← 深度 1                       │
│       ●     ●   ●     ●        ← 深度 2                       │
│ ...                                                           │
├ - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - ┤  ← 分隔线
│  ARRAY 区                                                     │
│  ● ● ● ● ● ● ●  （小尺寸小球，同步显示数组当前顺序）            │
│  0 1 2 3 4 5 6  （下标标签）                                   │
└──────────────────────────────────────────────────────────────┘
```

**节点坐标计算**（均匀层间距）：

```typescript
// 节点 idx 的深度
depth = floor(log2(idx + 1))
// 同层位置
posInLevel = idx - (2^depth - 1)
// Y 坐标：根在 TOP_PAD+r，末层在 TOP_PAD+treeH-r，线性插值
y = TOP_PAD + r + (depth / maxDepth) * (treeH - 2 * r)
// X 坐标：该层格子均分
cellW = xTotal / 2^depth
x = 40 + posInLevel * cellW + cellW / 2
```

**动态 Canvas 高度**：

```typescript
function getRequiredHeight(n: number): number {
  if (n <= 1) return TOP_PAD + 80 + BOT_PAD;
  const maxDepth = Math.floor(Math.log2(n));
  return TOP_PAD + (maxDepth + 1) * 90 + BOT_PAD;  // 每层最小 90px
}
```

`SortBarCanvasHeap.vue` 将 `getRequiredHeight(n)` 绑定到容器 `minHeight`，配合 `overflow-y: auto` 支持纵向滚动，避免层数多时树节点重叠。

**小球样式（霓虹光晕）**：

| 绘制层 | 效果 |
|-------|------|
| 第一次 fill + `shadowColor=border` | 外层彩色光晕（霓虹感） |
| 第二次 fill + `shadowOffsetY=3` | 底部投影（立体感） |
| stroke `border` 色 | 彩色描边 |
| 数字居中 | `bold` 字体，`textShadow` 防混淆 |

**小球颜色方案**（fill 主色，border 同色系浅色用于光晕）：

| 状态 | fill | 触发条件 |
|------|------|---------|
| default | 蓝 `#1e50a8` | 正常节点 |
| compare | 黄褐 `#8c6c00` | `comparing` 中 |
| pivot | 深紫 `#601f90` | sift-down 根（`pivot`） |
| swapping | 深红 `#901818` | 正在交换 |
| sorted | 绿 `#1a7810` | 最大堆已就位 |
| sortedMin | 青 `#0a7070` | 最小堆已就位 |
| pending | 暗紫 `#382070` | 当前堆范围（`groupIndices`） |
| outside | 近黑 `#111520` | 已排出堆外（预留） |

**swap 动画（`FlyTask`）**：

- 父子相邻节点（`2i+1` 或 `2i+2`）：**直线飞行**
- 跨层节点（根↔末尾等）：**弧线飞行**，控制点 `topY` 钳制到 `TOP_PAD/2` 以上，防止弧线超出画布

```typescript
function needsArc(i, j): boolean {
  return !(2*i+1 === j || 2*i+2 === j || 2*j+1 === i || 2*j+2 === i);
}
// 弧线控制点（防溢出）
const topY = Math.max(TOP_PAD / 2, Math.min(p1.y, p2.y) - arcHRaw);
```

**边（Edge）绘制**：每帧根据 `balls[i].x/y` 实时绘制父子连线，动画期间连线跟随小球实时更新。活跃边（compare/pivot 相关）加亮黄色高亮。

**`onStep` 接口**：

| 步骤类型 | 行为 |
|---------|------|
| `swap` | 启动 FlyTask（Promise），动画结束后 resolve |
| 其他 | 调用 `refreshColors()` 刷新颜色，不阻塞 |

## 7. 动画速度控制

### 速度参数传递链

```
ControlPanel 滑块（min: 20ms，max: 500ms）
    → sortStore.animationSpeed
    → 算法页面 props.animationSpeed
    → useSortAnimation.speed
    → canvasRef.applyStep(step)
    → 各渲染器 applyStep(step, animationSpeed)
    → flyBar() / wait() 的实际 duration 参数
```

**无硬性下限**：所有渲染器均直接使用 `animationSpeed` 作为实际时长，不做 `Math.max(speed, floor)` 限制。UI 滑块 `min="20"` 是唯一约束，用户设置多少就是多少。

### isAnimating 保护机制

`useSortAnimation` 维护 `isAnimating` 标志，在 Canvas 的 `await applyStep()` 期间为 `true`。

速度变化时（`watch(speed)`）若检测到 `isAnimating === true`，**跳过本次重启**，避免两个 `applyStep` 并发造成动画叠加。新速度在当前动画结束后的下一步自然生效。

```typescript
watch(speed, () => {
  if (localPlaying.value && currentStep.value < steps.value.length && !isAnimating) {
    stop(); play();
  }
});
```

### reset(regenerate) — 可选重新生成步骤

`reset` 支持可选参数 `regenerate`，用于需要在保持同一数组的情况下更换算法参数（如堆排序切换最大堆/最小堆）：

```typescript
function reset(regenerate = false) {
  stop();
  array.value = [...originalArray.value];
  if (regenerate && originalArray.value.length > 0) {
    steps.value = sortFn(originalArray.value.map(e => e.value));  // 重新生成步骤
    valueToDisplayIndex = buildValueToDisplayIndex();
  }
  currentStep.value = 0;
  // ...
}
```

调用方（HeapSort.vue）切换堆模式时调用 `reset(true)`，使 `sortFn` 闭包以新的 `heapMode.value` 重新计算步骤序列。

## 附录：高亮状态优先级

当多个状态同时命中时，渲染器按以下优先级着色：

```
pivot > swapping (动画足够表达) > comparing > pending > sorted > default
```

**特殊继承规则**：
- `swap` / `merge` 步骤：**继承**上一步的 `comparing`、`pending`、`pivot` 状态，避免"比较→交换"时颜色突变
- `merge-set` 步骤：不更新 `prevHighlightedIndices`，保持 pending 高亮区间持续显示
- 所有 `bucket-*` 步骤：主数组高亮全部置空（由渲染器内部独立管理颜色）
