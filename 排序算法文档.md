## 架构设计

### 核心设计思路：步骤预计算 + 异步动画同步

```
用户操作 → store.isPlaying 变化 → useSortAnimation 响应
    → 从预计算 steps 取当前步骤
    → 调用 canvasRef.applyStep(step) 驱动动画
    → await 动画完成后再推进下一步
```

**为什么这样做：**
1. **预计算步骤**：排序算法一次性生成所有步骤，存储于 `steps[]`，确保排序逻辑与渲染完全解耦
2. **异步动画同步**：通过 `async/await` 保证每一步动画播放完毕后才执行下一步，视觉上流畅且准确
3. **状态分层**：
   - `sortStore`：全局状态（originalArray、animationSpeed、arraySize、algorithm）
   - `useSortAnimation`：各算法组件私有状态（array、steps、currentStep、comparisons、swaps、isPlaying）
   - `useCanvasRenderer`：纯渲染状态（barStates、animationQueue）

### 组件层次

```
App.vue
├── ControlPanel.vue           # 操作入口（emit reset/step/new-array）
└── SortVisualizer.vue         # 动态组件切换
    └── [BubbleSort|InsertionSort|MergeSort|QuickSort|ShellSort]
        ├── SortBarCanvas.vue  # Canvas 渲染 + 动画执行
        └── stats-bar          # 统计信息（stats-left: 按钮+状态 | operation-item: 当前操作）
```

**stats-bar 布局**：
- `stats-left`：控制按钮（播放/暂停、单步、重置）+ 紧凑统计（比较/交换/步骤/gap）+ 状态指示器
- `operation-item`（固定右侧）：当前操作描述

**Corner brackets**：SortBarCanvas 使用 canvas-wrapper 结构包裹 canvas 和四个角落装饰 brackets，brackets 相对于 canvas 定位，与 canvas 四角精确对齐（之前相对容器定位导致偏移）。

### 状态指示器

4状态圆点动画，`statusText` 和 `statusClass` 由 `useSortAnimation` composable 统一提供（各算法组件通过 destructuring 获取，无需各自实现）：

| 状态 | class | 颜色 | 条件 |
|------|-------|------|------|
| 就绪 | `ready` | 灰色 `#8b95a8` | `currentStep === 0` |
| 播放中 | `playing` | 青色 `#5dddd4` + pulse 动画 | `localPlaying === true` |
| 已暂停 | `paused` | 红色 `#ff8a8a` | `!localPlaying && currentStep > 0 && currentStep < steps.length` |
| 已完成 | `done` | 绿色 `#6bff6b` | `currentStep >= steps.length` |

## 渲染机制

### Canvas 渲染循环

`useCanvasRenderer` 内部通过 `requestAnimationFrame` 驱动持续渲染循环：

```typescript
function startRenderLoop() {
  if (animationFrameId !== null) return;
  animationFrameId = requestAnimationFrame(draw); // 循环绘制
}
```

每帧执行：
1. `drawBackground()` — 绘制渐变背景 + 网格线
2. `processAnimations(timestamp)` — 更新动画队列中的柱状图位置
3. `barStates.forEach(drawBar)` — 绘制每根柱子

### 柱状图状态

```typescript
interface BarState {
  index: number;      // 在数组中的原始索引
  value: number;      // 数值（唯一标识，排序中不变）
  displayIndex: number; // 固定序号（1-based），跟随元素移动
  x, targetX: number; // 当前位置 & 目标位置
  y: number;          // y 坐标（动画时会上升形成弧线）
  width, height: number;
  color: { r, g, b }; // RGB 颜色
  glowIntensity: number; // 发光强度
}
```

**高度计算**：最小高度 5 像素，确保 value=1 时也能正常显示。
**数值标签**：bar 高度 >= 8px 时才显示，避免小柱子文字糊成一团。

### 高亮状态映射

通过 `highlightedIndices` computed 从当前步骤推导颜色：

| 状态 | 颜色 | 场景 |
|------|------|------|
| comparing | 黄色 `#FFCC00` | 正在比较的两个元素 |
| pending | 淡紫色 `#9575CD` | 当前排序组的元素（待排序） |
| sorted | 绿色 `#67C23A` | 已确定位置的元素 |
| pivot | 紫色 `#9B59B6` | 快速排序的基准元素 |
| swapping | 继承状态或默认 | 物理动画已足够表达交换动作 |

**交换步骤的特殊处理**：
- `swap` / `merge` 步骤会继承上一步的所有高亮状态（comparing、pending、sorted、pivot）
- 如果交换中的柱子上一步也是比较状态，保持黄色高亮
- 这样用户可以看到"从比较过渡到交换"的连续视觉反馈

**归并排序步骤的特殊处理**：
- `merge-set`：清除 comparing，保持 pending（groupIndices 合并区间），**不更新** `prevHighlightedIndices`
- `merge-back`：清空所有高亮（comparing / pending / swapping），待后续 sorted 步骤接手

**优先级**：pivot > (swapping + comparing 保持高亮) > comparing > pending > sorted > default

### 交换动画（swap / merge）

`animateSwap` 使用**抛物线弧度**模拟真实交换：

```typescript
const arcHeight = Math.sin(progress * Math.PI) * 50; // 0 → 最高 → 0
bar1.x = startX1 + deltaX * progress;
bar1.y = baseY - arcHeight; // 向上抛起
```

使用 `easeOutCubic` 缓动曲线使动画平滑。

### 播放控制逻辑

```typescript
// useSortAnimation.ts
watch(isPlaying, (playing) => playing ? play() : stop());

function play() { localPlaying.value = true; step(); }
async function step() {
  if (!localPlaying.value || currentStep.value >= steps.value.length) {
    localPlaying.value = false; return;
  }
  const delay = await applyStep(steps.value[currentStep.value]);
  timer = setTimeout(step, delay ?? speed.value);
}
```

`applyStep` 中 `await canvasRef.value?.applyStep(step)` 确保动画完成后才 `setTimeout(step, delay)` 进入下一步。

### 单步执行

`stepOnce()` 独立于播放循环，直接执行当前步骤：
```typescript
async function stepOnce() {
  if (!localPlaying.value && currentStep.value < steps.value.length)
    await applyStep(steps.value[currentStep.value]);
}
```

## 核心类型定义

```typescript
// types/sorting.ts
type StepType =
  | "compare"
  | "swap"
  | "merge"
  | "set"
  | "sorted"
  | "pivot"
  | "merge-set"   // 归并排序：元素从上排飞入下排辅助区
  | "merge-back"; // 归并排序：下排所有元素整体飞回上排

interface SortStep {
  type: StepType;         // 步骤类型
  indices: number[];      // 涉及的数组索引
  description: string;    // 步骤描述（显示在 stats-bar）
  arraySnapshot?: number[]; // 数组快照（swap 为交换前，merge-back 为合并后）
  gap?: number;           // 步长（希尔排序用）
  groupIndices?: number[]; // 当前排序组的全部索引（归并排序合并区间）
  tempSnapshot?: (number | null)[]; // 辅助数组快照（归并排序下排专用，目前预留）
}

type SortAlgorithm = "bubble" | "insertion" | "merge" | "quick" | "shell";
```

## 排序算法详解

### 冒泡排序 (Bubble Sort)

**思想**：相邻元素两两比较，大值不断"冒泡"到数组末端。每轮遍历将当前最大值移动到正确位置。

**核心代码**：
```typescript
// sortingAlgorithms.ts
export function bubbleSort(arr: number[]): SortStep[] {
  const steps: SortStep[] = [];
  const n = arr.length;
  const array = [...arr];

  for (let i = 0; i < n - 1; i++) {           // 外循环：轮数
    for (let j = 0; j < n - i - 1; j++) {     // 内循环：相邻比较
      steps.push(createStep("compare", [j, j + 1],
        `比较 arr[${j}]=${array[j]} 和 arr[${j + 1}]=${array[j + 1]}`));

      if (array[j] > array[j + 1]) {           // 逆序则交换
        steps.push(createStep("swap", [j, j + 1],
          `交换 arr[${j}] 和 arr[${j + 1}]`, [...array]));
        [array[j], array[j + 1]] = [array[j + 1], array[j]];
      }
    }
    steps.push(createStep("sorted", [n - i - 1], // 标记已排序
      `arr[${n - i - 1}]=${array[n - i - 1]} 已排序`, [...array]));
  }
  return steps;
}
```

**示例**：`[5, 3, 8, 1, 9, 4, 7, 2, 6]`
- 第 1 轮：`[3, 5, 1, 2, 6, 4, 7, 8, 9]` — 9 冒泡到末端
- 第 2 轮：`[3, 1, 2, 5, 6, 4, 7, 8, 9]` — 8 就位
- 第 3 轮：`[1, 2, 3, 4, 5, 6, 7, 8, 9]` — 7 就位
- 依此类推... 直至全部有序

**复杂度**：O(n²) 时间，O(1) 空间

### 插入排序 (Insertion Sort)

**思想**：将数组分为"有序区"和"无序区"，逐个将无序区元素插入有序区的正确位置，像整理扑克牌。实现上使用真正的 swap 动画来模拟 key 与左元素交换直到到达正确位置。

**核心代码**：
```typescript
export function insertionSort(arr: number[]): SortStep[] {
  const steps: SortStep[] = [];
  const array = [...arr];
  const n = arr.length;

  // 初始：第一个元素作为有序区起点
  steps.push(createStep("sorted", [0],
    `arr[0]=${array[0]} 已排序（作为初始有序区）`, [...array]));

  for (let i = 1; i < n; i++) {
    const key = array[i];
    let j = i - 1;

    steps.push(createStep("compare", [i],
      `将 arr[${i}]=${key} 插入有序区`, [...array]));

    // 从有序区末尾向左比较，用真正的 swap 模拟 key 与左元素交换
    while (j >= 0 && array[j] > key) {
      steps.push(createStep("compare", [j, j + 1],
        `比较 arr[${j}]=${array[j]} 和 key=${key}`, [...array]));
      // 用真正的 swap 模拟 key 与左元素交换（实现右移效果）
      steps.push(createStep("swap", [j, j + 1],
        `交换 arr[${j}]=${array[j]} 和 arr[${j + 1}]=${key}`, [...array]));
      [array[j], array[j + 1]] = [array[j + 1], array[j]];
      j--;
    }
    // 循环结束后 key 已在正确位置，无需额外插入
  }
  return steps;
}
```

**示例**：`[5, 3, 8, 1, 9, 4, 7, 2, 6]`
- 有序区 `[5]`，处理 3：交换 5 和 3 → `[3, 5, 8, 1, 9, 4, 7, 2, 6]`
- 处理 8：在 5 后，无需移动 → `[3, 5, 8, 1, 9, 4, 7, 2, 6]`
- 处理 1：依次交换 8、5、3 → `[1, 3, 5, 8, 9, 4, 7, 2, 6]`
- 处理 9：在 8 后，无需移动 → `[1, 3, 5, 8, 9, 4, 7, 2, 6]`
- 处理 4：依次交换 8、5、3 → `[1, 3, 4, 5, 8, 9, 7, 2, 6]`
- 依此类推... 直至 `[1, 2, 3, 4, 5, 6, 7, 8, 9]`

**复杂度**：O(n²) 时间，O(1) 空间

### 归并排序 (Merge Sort)

**思想**：分治策略。将数组递归分割为单个元素，再自底向上合并有序子数组。合并时两路归并。

**双排动画设计**：归并排序使用专用的双排 Canvas（`SortBarCanvasMerge`）。上排是主数组，下排是合并临时区。合并过程中胜出元素逐个"飞入"下排，合并完成后下排元素整体"飞回"上排。

**步骤序列**：
```
compare[left, mid, right]              ← 分割通知（无动画）
compare[left, right] + groupIndices    ← 开始合并，上排高亮整个合并区间
loop:
  compare[i, j] + groupIndices         ← 比较左右指针（黄色高亮）
  merge-set[source, dest]              ← 胜出元素从上排飞入下排
merge-back[mergeRange] + arraySnapshot ← 下排整体飞回上排（附合并后快照）
sorted[全部]                           ← 完成
```

**核心代码**：
```typescript
export function mergeSort(arr: number[]): SortStep[] {
  const steps: SortStep[] = [];
  const array = [...arr];

  function merge(left: number, mid: number, right: number) {
    const leftArr = array.slice(left, mid + 1);
    const rightArr = array.slice(mid + 1, right + 1);
    const mergeRange = Array.from({ length: right - left + 1 }, (_, idx) => left + idx);
    const temp: number[] = new Array(right - left + 1);
    let i = 0, j = 0, k = left;

    // 通知画布：上排高亮整个合并区间
    steps.push(createStep("compare", [left, right],
      `合并区间 [${left}, ${mid}] 和 [${mid + 1}, ${right}]`,
      [...array], undefined, mergeRange));

    while (i < leftArr.length && j < rightArr.length) {
      steps.push(createStep("compare", [left + i, mid + 1 + j],
        `比较 leftArr[${i}]=${leftArr[i]} 和 rightArr[${j}]=${rightArr[j]}`,
        [...array], undefined, mergeRange));

      if (leftArr[i] <= rightArr[j]) {
        temp[k - left] = leftArr[i];
        // indices[0]=源索引（上排位置），indices[1]=目标列（下排输出位置）
        steps.push(createStep("merge-set", [left + i, k],
          `${leftArr[i]} 胜出，飞入下排第 ${k} 槽`,
          [...array], undefined, mergeRange));
        i++;
      } else {
        temp[k - left] = rightArr[j];
        steps.push(createStep("merge-set", [mid + 1 + j, k],
          `${rightArr[j]} 胜出，飞入下排第 ${k} 槽`,
          [...array], undefined, mergeRange));
        j++;
      }
      k++;
    }
    // 剩余元素直接飞入下排
    while (i < leftArr.length) { temp[k - left] = leftArr[i];
      steps.push(createStep("merge-set", [left + i, k], `剩余 ${leftArr[i]} 飞入下排`,
        [...array], undefined, mergeRange));
      i++; k++; }
    while (j < rightArr.length) { temp[k - left] = rightArr[j];
      steps.push(createStep("merge-set", [mid + 1 + j, k], `剩余 ${rightArr[j]} 飞入下排`,
        [...array], undefined, mergeRange));
      j++; k++; }

    // 复写回主数组，通知画布：下排整体飞回上排
    for (let m = left; m <= right; m++) array[m] = temp[m - left];
    steps.push(createStep("merge-back", mergeRange,
      `区间 [${left}, ${right}] 合并完成，全部归位`, [...array]));
  }

  function sort(left: number, right: number) {
    if (left >= right) return;
    const mid = Math.floor((left + right) / 2);
    steps.push(createStep("compare", [left, mid, right],
      `分割区间 [${left}, ${right}]，中点为 ${mid}`, [...array]));
    sort(left, mid);
    sort(mid + 1, right);
    merge(left, mid, right);
  }

  sort(0, array.length - 1);
  steps.push(createStep("sorted",
    Array.from({ length: array.length }, (_, i) => i), `排序完成`, [...array]));
  return steps;
}
```

**关键设计**：
- `merge-set` 不携带 `arraySnapshot`（主数组未变，避免触发 `updateBars` 清空飞行中的下排柱子）
- `merge-back` 携带合并后完整快照，触发主数组重建
- 上排飞走的位置渲染为"幽灵"（12% 透明度虚线描边），合并完成后消失

**示例**：`[5, 3, 8, 1, 9, 4, 7, 2, 6]`
- 分割：`[5, 3, 8, 1]` `[9, 4, 7, 2, 6]`
- 再分：`[5, 3]` `[8, 1]` `[9, 4]` `[7, 2, 6]`
- 细分为：`[5]` `[3]` `[8]` `[1]` `[9]` `[4]` `[7]` `[2]` `[6]`
- 合并：`[3, 5]` `[1, 8]` `[4, 9]` `[2, 6, 7]`
- 继续归并：`[1, 3, 5, 8]` `[2, 4, 6, 7, 9]`
- 最终：`[1, 2, 3, 4, 5, 6, 7, 8, 9]`

**复杂度**：O(n log n) 时间，O(n) 空间

### 快速排序 (Quick Sort)

**思想**：分治策略。选择基准（pivot），将数组分为小于基准和大于基准两部分，递归排序两部分。

**核心代码**：
```typescript
export function quickSort(arr: number[]): SortStep[] {
  const steps: SortStep[] = [];
  const array = [...arr];

  function partition(low: number, high: number): number {
    const pivot = array[high];
    steps.push(createStep("pivot", [high],
      `选择 arr[${high}]=${pivot} 作为基准`, [...array]));

    let i = low - 1;
    for (let j = low; j < high; j++) {
      steps.push(createStep("compare", [j, high],
        `比较 arr[${j}]=${array[j]} 和 pivot=${pivot}`, [...array]));

      if (array[j] < pivot) {
        i++;
        if (i !== j) {
          steps.push(createStep("swap", [i, j],
            `交换 arr[${i}]=${array[i]} 和 arr[${j}]=${array[j]}`, [...array]));
          [array[i], array[j]] = [array[j], array[i]];
        }
      }
    }
    // 将基准放到正确位置
    if (i + 1 !== high) {
      steps.push(createStep("swap", [i + 1, high],
        `将基准放到正确位置 arr[${i + 1}]=${array[i + 1]}`, [...array]));
      [array[i + 1], array[high]] = [array[high], array[i + 1]];
    }
    steps.push(createStep("sorted", [i + 1],
      `arr[${i + 1}]=${array[i + 1]} 已在正确位置`, [...array]));
    return i + 1;
  }

  function sort(low: number, high: number) {
    if (low >= high) {
      if (low === high)
        steps.push(createStep("sorted", [low],
          `arr[${low}]=${array[low]} 已排序`, [...array]));
      return;
    }
    const pi = partition(low, high);
    sort(low, pi - 1);
    sort(pi + 1, high);
  }

  sort(0, array.length - 1);
  return steps;
}
```

**示例**：`[5, 3, 8, 1, 9, 4, 7, 2, 6]`，选 `6` 为基准
- 基准 6：整理后 → `[5, 3, 4, 2, 6, 8, 9, 7, 1]`（6 到正确位置）
- 左 `[5, 3, 4, 2]` 选 `2` → `[2, 3, 4, 5]`（2 到正确位置）
- 右 `[8, 9, 7, 1]` 选 `1` → `[1, 7, 8, 9]`（1 到正确位置）
- 递归继续... 直至 `[1, 2, 3, 4, 5, 6, 7, 8, 9]`

**复杂度**：平均 O(n log n)，最坏 O(n²)，O(log n) 空间

### 希尔排序 (Shell Sort)

**思想**：希尔排序是插入排序的改进版本，通过设置间隔序列（gap）逐步缩小排序范围。先将数组分为多个子数组分别排序，最后使用插入排序对整体排序。

**核心代码**：
```typescript
export function shellSort(arr: number[]): SortStep[] {
  const steps: SortStep[] = [];
  const a = [...arr];
  const n = a.length;

  for (let gap = Math.floor(n / 2); gap > 0; gap = Math.floor(gap / 2)) {
    for (let i = gap; i < n; i++) {
      // 计算当前 i 所属的组：所有 ≡ i (mod gap) 的索引
      const group: number[] = [];
      const remainder = i % gap;
      for (let k = remainder; k < n; k += gap) group.push(k);
      let j = i;
      while (j >= gap) {
        steps.push(createStep("compare", [j - gap, j],
          `比较间隔 ${gap} 内的元素 [${j - gap}]=${a[j - gap]} 和 [${j}]=${a[j]}`,
          undefined, gap, group));
        if (a[j - gap] > a[j]) {
          steps.push(createStep("swap", [j - gap, j],
            `交换 arr[${j - gap}]=${a[j - gap]} 和 arr[${j}]=${a[j]}`,
            [...a], gap, group));
          [a[j - gap], a[j]] = [a[j], a[j - gap]];
          j -= gap;
        } else {
          break;
        }
      }
    }
  }

  const sortedIndices = Array.from({ length: n }, (_, i) => i);
  steps.push(createStep("sorted", sortedIndices, "排序完成"));
  return steps;
}
```

**示例**：`[5, 3, 8, 1, 9, 4, 7, 2, 6]`，初始 gap = 4
- gap=4：分组 `[5,9]`, `[3,4]`, `[8,7]`, `[1,2]`, `[6]` 分别插入排序
- gap=2：分组 `[5,8,9,7]`, `[3,1,4,2]`, `[6]` 分别排序
- gap=1：最后进行插入排序
- 最终：`[1, 2, 3, 4, 5, 6, 7, 8, 9]`

**复杂度**：O(n log² n) 时间，O(1) 空间

## 算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 | 适用场景 |
|------|-----------|-----------|--------|---------|
| 冒泡 | O(n²) | O(1) | 稳定 | 教学、小规模数据 |
| 插入 | O(n²) | O(1) | 稳定 | 基本有序、数据较少 |
| 希尔 | O(n log² n) | O(1) | 不稳定 | 中等规模数据 |
| 归并 | O(n log n) | O(n) | 稳定 | 大数据、外部排序 |
| 快速 | O(n log n) | O(log n) | 不稳定 | 通用场景、高性能 |

```typescript

  const [i, j] = step.indices;
  finalSnapshot = [...step.arraySnapshot];
  [finalSnapshot[i], finalSnapshot[j]] = [finalSnapshot[j], finalSnapshot[i]];
}
```

**displayIndex round-robin 分配**（处理重复值）：快照重建时按出现顺序依次分配 displayIndex，防止相同值的柱子序号混淆：
```typescript
// 示例：originalArray=[3,1,3,3]，displayIndex=[0,1,2,3]
// 快照 [3,3,1,3] → displayIndex 依次分配 [0,2,1,3]
const roundRobin = new Map<number, number>();
finalSnapshot.map(value => {
  const available = valueToDisplayIndices.get(value); // e.g. [0,2,3]
  const idx = roundRobin.get(value) ?? 0;
  const displayIndex = available[idx % available.length];
  roundRobin.set(value, idx + 1);
  return { value, displayIndex };
});
```
